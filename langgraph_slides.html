核心概念

📋 State（状态）

在图中流转的共享数据容器，就像一个所有节点都能读写的"共享记事本"。

class GreetingState(TypedDict): name: str # 用户的名字 greeting: str # 问候语

⚙️ Node（节点）

一个处理数据的 Python 函数，接收 State，返回要更新的字段。

def greet(state: GreetingState) -> dict: name = state["name"] return {"greeting": f"你好，{name}！"}

🔗 Edge（边）

节点之间的连接线，定义执行顺序。

graph.add_edge(START, "greet") graph.add_edge("greet", "add_emoji") graph.add_edge("add_emoji", END)

简单聊天机器人

📝 Annotated —— 追加模式

普通字段是覆盖模式，但聊天需要追加消息：

模式行为适用场景
普通字段新值覆盖旧值greeting, name
Annotated新值追加到列表messages（对话历史）
class ChatState(TypedDict): # 消息列表，使用追加模式（不会覆盖旧消息） messages: Annotated[list, add_messages]

🤖 ChatOpenAI

LangChain 对 OpenAI 的封装，处理 API 调用细节。

# 创建 LLM 实例 llm = ChatOpenAI( model="gpt-4o-mini", # 快速便宜 temperature=0.7, # 创造性 ) # 节点函数 def chatbot(state): response = llm.invoke(state["messages"]) return {"messages": [response]}

💬 消息类型

HumanMessage — 用户发的消息

AIMessage — AI 的回复

ToolMessage — 工具执行结果（第四课学）

LLM 通过消息类型区分"谁说了什么"
消息越多 = 上下文越丰富（也越费钱）
对话历史是 LLM "记忆"的来源

条件边与分支逻辑

🔀 条件边 vs 普通边

类型目标方法
普通边固定的下一个节点add_edge(A, B)
条件边由路由函数动态决定add_conditional_edges(A, fn)
# 路由函数：返回下一个节点的名称 def route_to_department(state) -> str: if state["category"] == "technical": return "technical_support" elif state["category"] == "billing": return "billing_support" return "general_support"

工具调用 & ReAct 智能体

🛠️ @tool 装饰器

把普通函数变成 LLM 可调用的工具。

# LLM 通过 docstring 判断何时使用此工具 @tool def add(a: float, b: float) -> float: """把两个数加在一起""" return a + b # 绑定工具给 LLM llm_with_tools = llm.bind_tools([add, multiply])

📖 执行过程示例

第 1 轮："3+5=?" → LLM 说 "调用 add(3,5)" → 返回 8
第 2 轮：LLM 说 "调用 multiply(8,4)" → 返回 32
第 3 轮：LLM 说 "答案是 32" → 到达终点

这就是所有 AI Agent 的基础

ChatGPT 插件、Claude 工具调用、
所有能搜索/计算/操作的 AI 助手

检查点与记忆

💾 Checkpointer 检查点

每步执行后自动保存状态，支持断点续传和多用户。

# 只需两行代码即可启用记忆！ memory = MemorySaver() graph = builder.compile(checkpointer=memory)

🧵 thread_id 线程隔离

不同 thread_id 代表不同的对话，互不干扰。

# Alice 的对话 config_a = {"configurable": {"thread_id": "alice"}} graph.invoke(msg, config=config_a) # Bob 的对话（完全隔离） config_b = {"configurable": {"thread_id": "bob"}} graph.invoke(msg, config=config_b)

🏭 生产环境持久化选择

🧪

存在 SQLite

轻量持久化

🐘

PostgresSaver

存在 PostgreSQL
多服务器部署
生产推荐

人机协作 (Human-in-the-Loop)

⏸️ interrupt_before —— 暂停机制

在关键节点之前暂停，让人类审核后再继续。

graph = builder.compile( checkpointer=memory, interrupt_before=["tools"], # 执行工具前暂停 ) # 查看 AI 想做什么 state = graph.get_state(config) # 批准后恢复执行 graph.invoke(None, config=config)

✅ 审批流程

→

→

→

批准 → invoke(None) 继续执行

拒绝 → update_state() 注入拒绝消息

📧 发送邮件审批 💳 付款确认 🗄️ 数据库写入审核 📄 内容发布审核 🔒 合规审计

📋 共享状态 = 协作白板

class TeamState(TypedDict): messages: Annotated[list, add_messages] task: str # 任务描述 research: str # 研究员写入 draft: str # 写手写入 review: str # 审核员写入 next_agent: str # 主管决定

研究员写 research → 写手读它写 draft → 审核员审 draft 写 review
